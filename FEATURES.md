# FEATURES.md

## 目标功能清单

本项目长期目标是形成一个可工程化运行的 **OpenAI 兼容 API / 账号池管理 / 注册机供给接入 / 自动化运维系统**。

---

## 核心目标能力

1. **OpenAI 兼容 API 网关**
   - 暴露统一请求入口
   - 支持模型列表与调用转发

2. **账号池管理**
   - 账号录入
   - 导入外部注册机产出
   - 账号状态管理
   - 账号健康检查

3. **注册机供给接入**
   - 将 `auto_reg` 作为主注册机项目
   - 对接注册结果导入
   - 对接状态回写
   - 为后续指纹浏览器执行器接入预留接口

4. **调度与路由**
   - 请求路由
   - 限流/配额管理
   - 基础负载均衡
   - 可用账号优先选择

5. **自动化控制**
   - 定时巡检
   - 补号触发
   - 告警通知
   - 审计日志

6. **部署与运维**
   - Ubuntu + Docker 部署
   - 反向代理与 HTTPS
   - 配置管理
   - 运行状态查看

---

## 近期不应误判为已完成的能力

以下内容目前仍主要停留在方案层或骨架层，不应视为已实现：

- 指纹浏览器 API 已完成接入
- 导入账号已参与真实路由选择
- 完整健康检查与自动补号闭环
- 真正可运行的动态调度系统
- 完整监控告警系统
