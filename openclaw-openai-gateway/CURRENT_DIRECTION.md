# 当前方向（openclaw-openai-gateway）

当前方向不是继续做“第三方 provider 接入平台”，而是先把 **能稳定反代出真实额度** 这件事打通。

你已经明确了主线：

1. **先做额度反代**，不是先做 provider 花活
2. 反代来源优先是：
   - **Codex App**
   - **Web 会话 / Web 额度**
3. 先把可用额度池做起来，再谈：
   - 对外提供 API Key
   - 账号池
   - 容灾
   - 平衡
   - 调度

一句话：

**先解决“怎么稳定反代出额度”，再解决“怎么把额度组织成池并对外卖 API Key”。**
