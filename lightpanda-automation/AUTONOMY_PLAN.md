# AUTONOMY_PLAN.md

项目自动推进规则草案。

## 周期执行规则

- 每轮先读：`ROADMAP.md` / `STATUS.md` / `TODO.md` / `RUN_STATE.json`
- 每轮先更新 roadmap（过去 / 现在 / 未来）
- 每轮做一个最小但真实的推进动作
- 每轮执行后写 `EXECUTION_LOG.md`
- 偶数轮：做 bug 检查与功能验证
- 每 4 轮：输出一次阶段汇总

## 推进动作范围

包括但不限于：
- 借鉴开源项目
- 完善工程文档
- 写代码
- 改代码
- 设计接口
- 设计数据库 schema
- 实现 fake runner
- 做功能验证
- 发现和记录 bug

## 约束

- 不要每轮做过重的大动作
- 优先做小步、可验证、可回滚的推进
- 所有推进都必须回写文档与状态文件
